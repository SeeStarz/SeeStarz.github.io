Linux

I use arch btw. Need I say more? Anyways I started using arch a few months before my national exam announcment. Having partitioned, installed, and configuring my system with the help of the arch wiki I learned a lot of new things. For example the UEFI booting system has it's own EFI partition that contains programs that points to other programs in other partition. How the package manager handles dependencies in linux is vastly different than how it works in windows, it's way more transparent in saying what should and shouldn't conflict with each other.

I just love working in the terminal. Okay not the tty, that is pretty boring, although usable. Terminal emulators though, it can display pretty colors and symbols while still being keyboard oriented. I'm not saying I know and are good with every single cli tools, but I very much like them. Git, gio, magick, gifsickle, ffmpeg, unison, pacman, yay, pipx, qpdf, and others. These cli tools has the advantage that it is easily automatable. Tui apps are also very cool, like nvim, nchat, tiptop, and so on.

Typing

Using the colemak layout I clocked in at average 100 wpm on typeracer. With the qwerty layout around 80wpm which is still good, but I don't like typing in qwerty. Using keyboard instead of mouse is beneficial. No need to change focus to aim and click at something, just press some keys. The difference might be only half a second but the interruption will slow down the train of thought. Also using mostly keyboard eliminates the need of a mouse, making it much more comfortable to work anywhere. I used neovim as my main code editor and used hyprland as my main compositor. Both of these provide a very quick shortcut to minimize delay and focus on action.
